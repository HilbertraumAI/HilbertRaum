## How this was done

Five independent reviewers audited the codebase from distinct viewpoints, each verifying claims
against the actual source rather than the docs:

1. **Security auditor** — crypto/vault, offline guarantee, Electron hardening, IPC trust boundary, XSS.
2. **Correctness/reliability engineer** — races, error handling, RAG/runtime/DB edge cases.
3. **Docs-vs-code auditor** — contradictions between `docs/`/`BUILD_STATE.md` and the code.
4. **UX & accessibility auditor** — a11y, i18n in the UI, error/loading states, design-guideline fit.
5. **Staff engineer** — architecture, coupling, type safety, testing, maintainability, deps.

The headline HIGH findings (H1–H4 below) were re-verified by hand at the cited lines before
publishing.

## Overall assessment

**This is a strong, disciplined codebase.** Notable strengths the auditors independently confirmed:

- **Crypto/vault is genuinely solid:** AES-256-GCM with per-call random IVs (no reuse), Argon2id
  KDF with attacker-param DoS bounds, `timingSafeEqual` verifier, key zeroing on lock/unwrap/rewrap,
  a journaled atomic v1→v2 password-change migration with crash-recovery, startup plaintext-shred sweep.
- **Electron hardening is exemplary:** `contextIsolation`/`sandbox`/`nodeIntegration:false`, strict
  prod CSP (`script-src 'self'`, no inline/eval), deny-by-default `setWindowOpenHandler`/`will-navigate`,
  scoped audio-only permission handler. No XSS path via model output (react-markdown, no `rehype-raw`).
- **No hosted-AI calls, no telemetry, loopback-only sidecars** — the spec §0 hard rules hold.
- **Swappable-service architecture is textbook:** runtime/embedder/reranker/transcriber/OCR all sit
  behind narrow interfaces with DI seams; effectively zero `any`; uniformly guarded DB-column JSON parsing;
  zero TODO/FIXME/HACK debt; ~82 automated test files covering the spec-named invariants
  (never-throws, no-orphan runtime, GPU graceful degradation, vault data-safety).

No **CRITICAL** issues were found. The findings worth acting on are concentrated in a handful of
**permanent session-wide wedges** (H1, H4), **silent retrieval failure modes** (H2, H3), some
**accessibility/announcement** gaps, and **doc/script drift**.

## HIGH — fix these

### H1. ✅ FIXED — `importDocuments` leaks the vault document-work lease on a synchronous throw → password change wedged for the session
`apps/desktop/src/main/ipc/registerDocsIpc.ts:127-129`

`beginDocumentWork()` is called (line 127), then `expandPaths(paths)` (128) and
`createQueuedDocument(...)` per file (129) run **outside any try**. The `try/finally { releaseDocWork() }`
only begins inside the background IIFE (line 172-175). If `createQueuedDocument` throws (DB locked/closed
in a race, disk-full INSERT, constraint), the invoke throws and `releaseDocWork()` never runs.

**Impact:** `docWork` stays incremented forever; `changeWorkspacePassword` then permanently throws
`VaultBusyError` ("Documents are still being imported…") for the rest of the session, with no import
in progress to explain it. Restart-only recovery.

**Fix:** Wrap the synchronous setup (lines 128-139) in a `try` that calls `releaseDocWork()` on a
synchronous failure, or acquire the lease immediately before the IIFE.

### H2. ✅ FIXED — RAG context-token budget undercounts → grounded prompt can overflow the model window
`apps/desktop/src/main/services/rag/index.ts` (the `maxContextTokens` trim loop, ~L251-261)

The budget loop sums `approxTokenCount(c.text)` — a whitespace **word** count — and compares it
directly against `settings.ragMaxContextTokens` (model **tokens**) with **no** words→tokens factor.
Everywhere else in the codebase the conversion uses a 1.3 factor for exactly this
(`doctasks.ts` `SUMMARY_TOKENS_PER_WORD = 1.3`). Real BPE tokens run ~1.3–1.4× the word count, so the
assembled excerpt block can exceed the budget by ~30-40%.

**Impact:** The grounded prompt overflows; the runtime silently truncates, dropping excerpts (or the
trailing rules/question) → degraded or wrong document answers. Inconsistent with the conservative
budgeting the doc-task engine already applies.

**Fix:** Apply the same 1.3 factor against the limit, matching `SUMMARY_TOKENS_PER_WORD`.

### H3. ✅ FIXED — One corrupt/truncated embedding row throws and aborts the *entire* document query
`apps/desktop/src/main/services/embeddings/index.ts:144-145` (consuming `decodeVector`, L54-56)

The search loop guards a **`dimensions`-column** mismatch (`row.dimensions !== queryVector.length`,
line 144) and `continue`s — but it does **not** guard a `vector_blob` that is physically shorter than
`row.dimensions * 4` bytes (truncation, partial write). In that case `decodeVector` slices to
`dimensions*4` bytes, gets fewer, and `new Float32Array(buffer, 0, dimensions)` throws `RangeError`,
which propagates out of `search()` and aborts all retrieval.

**Impact:** A single bad row breaks *all* document search instead of skipping that chunk. Not in
`known-limitations.md`.

**Fix:** In the loop, skip + log when `row.vector_blob.length < row.dimensions * 4` (or wrap the
`decodeVector`/`cosineSimilarity` pair in a try/continue), mirroring the dimension-mismatch guard.

### H4. ✅ FIXED — Hidden-window PDF rasterizer permanently wedges if `new BrowserWindow` throws
`apps/desktop/src/main/services/ocr/rasterizer.ts:56-58`

`inUse = true` (line 56) is set **before** `new BrowserWindow(...)` (line 58); the `try/finally
{ inUse = false }` only starts after window creation and listener wiring. If `BrowserWindow`
construction throws (resource exhaustion, mid-quit, GPU-process crash), `inUse` stays `true` forever.

**Impact:** Every subsequent OCR run for the session fails with "PDF rasterizer is busy" until restart.

**Fix:** Move `inUse = true` inside the `try`, or reset it on a window-construction failure.

### H5 (docs). `build-commercial-drive.ps1` hardcodes the runtime build matrix that already has a source of truth
`scripts/build-commercial-drive.ps1:222-229`

A literal 6-row `{family;dir;backend;bin}` table (win=vulkan, win/cpu, mac=metal, linux=vulkan,
linux/cpu, whisper win=cpu) duplicates `model-manifests/runtime-sources.yaml` (`builds[].extract_to`/
`backend`), which the canonical `commercial-drive.ts assertCommercialDrive` reads **dynamically** via
`planRuntimeDownload`. The script's own comment says "keep them in sync."

**Impact:** If a default backend or `extract_to` changes in the YAML, the TS gate updates automatically
but the PS1 cross-check keeps asserting the stale matrix — it can pass a drive the real gate would fail
(or vice-versa), and the `.sh` twin may already diverge. The worst config-drift surface, guarded by a
comment only.

**Fix:** Parse the matrix out of the YAML (the script already block-parses the version), or shell into
a tiny node invocation of `assertCommercialDrive`. More broadly, add **one** vitest test that
regex-extracts the script literals (dir arrays, the two config JSONs, `drive_format_version`, this build
matrix, the sha/runtime/format rules) and asserts equality with the canonical TS + `runtime-sources.yaml`.
Pure string parsing — runs in CI where Node exists, no shell execution needed. This is the single
highest-leverage maintainability fix.

## MEDIUM

### Security / robustness

- **M-S1. Offline guard only LOGS, never blocks** — `offlineGuard.ts:64-84`, wired `index.ts:348-358`.
  `installOfflineNetworkGuard` monkey-patches `net.Socket.prototype.connect`, calls `onViolation`
  (warn + audit), then **always calls the original connect**. It's a tripwire, not enforcement; the
  offline guarantee rests entirely on "no remote-calling code exists" + the CSP. This is a conscious
  trade-off (blocking could break loopback IPC/sidecar), but it is feasible to **reject non-loopback
  connects while offline** since `isLoopbackHost` already exempts the sidecar. Either upgrade to
  enforce-on-violation or record the detection-only decision explicitly in `security-model.md`.

- **M-S2. IPC handlers trust renderer-supplied argument types** — `registerWorkspaceIpc.ts:79`,
  `registerDocsIpc.ts:122`, others. Args are TS-typed but not runtime-validated. `createWorkspace`
  evaluates `password.length` **outside** its try/catch (line 79), so a non-string password is an
  unhandled `TypeError` at the IPC boundary instead of a clean `{ ok:false }`. `importDocuments` passes
  `paths` to `expandPaths` without checking elements are strings (and accepts arbitrary machine paths,
  not just picker output). Several handlers do validate well (chat `mode` enum, dictation `instanceof
  Uint8Array` + size) — the pattern is known but applied inconsistently. Add small per-handler shape
  guards; the renderer is the untrusted boundary in Electron.

- **M-S3. ✅ FIXED — OCR hidden window lacks its own `setWindowOpenHandler`/`will-navigate`** —
  `rasterizer.ts:58-69` (fixed alongside H4). `webPreferences` are correct and channel isolation is good
  (`event.sender === win.webContents`), and the shared-session CSP applies, but this is the one place
  untrusted PDF bytes are rendered, and it has no window-level navigation/window-open guards like the
  main window. Add `setWindowOpenHandler(() => ({action:'deny'}))` + a `will-navigate` preventDefault.

- **M-C1. Post-ready sidecar `'error'` (without `'exit'`) skips the GPU crash auto-fallback** —
  `runtime/sidecar.ts:256-273, 340-368`. `child.once('error', …)` only records `spawnError` (consulted
  in `waitForHealthy`); after start, an `'error'` without `'exit'` never fires `onUnexpectedExit`
  (gated on `this.exited`), so the documented crash auto-fallback (architecture.md GPU record §5.3)
  doesn't fire and `stop()` (which awaits only `'exit'`) can hang. Fire `onUnexpectedExit` from the
  `'error'` handler when ready && !stopping; resolve `stop()`'s wait on `'error'` too.

- **M-C2. `stop()` returns without SIGKILL escalation when `child.kill()` throws → orphan sidecar** —
  `runtime/sidecar.ts:350-354`. On the kill-throw path `stop()` returns early without racing exit/grace
  or escalating to SIGKILL, leaving an orphan llama-server holding VRAM + the port. Still race exit and
  attempt SIGKILL on that path.

- **M-C3. GPU crash auto-fallback self-disables if `restart` throws synchronously** —
  `runtime/factory.ts:284-298`. `restarting = true` then `void deps.restart(opts).catch().finally(...)`.
  A synchronous throw from `restart` escapes before the `.finally` is built, so `restarting` stays
  `true` forever and all future auto-fallbacks are suppressed. Wrap in `try { … } catch { restarting = false }`.

- **M-C4. RRF tiebreak systematically suppresses keyword-only (exact-term) hits** —
  `rag/hybrid.ts:110, 118-120`. Keyword-only hits get `vectorRank = MAX_SAFE_INTEGER`; the tiebreak
  `rrfScore desc || vectorRank asc` means a #1 keyword-only chunk always loses the RRF-score tie to a
  #1 vector chunk — deprioritizing exact-term matches (invoice numbers, codes), the precise case hybrid
  search exists to catch. Tiebreak on the best individual rank across both lists.

- **M-C5. Retrieval (query embed / rerank) ignores the caller's abort signal** —
  `embeddings/e5.ts:160`, `reranker/llama.ts:159`. Both use `AbortSignal.timeout(...)` only;
  `opts.signal` is threaded into `chatStream` but not into `retrieve`/`embed`/`rerank`. A user "Stop"
  during query embedding or the ~24.7 s CPU rerank (a known limitation) is ignored until completion.
  Plumb `opts.signal` through, combined via `AbortSignal.any([signal, AbortSignal.timeout(...)])`.

### UX / accessibility

- **M-U2. "Stop" during streaming gives no interruption feedback** — `chat/ChatScreen.tsx:338-339,

- **M-U4. Two "Local · Offline" indicators can disagree** — sidebar `App.tsx:213` (refreshed only on

- **M-U5. Number/size formatting bypasses locale in spots** — `ModelsScreen.tsx:374-378`
  (`{gb} GB` raw), `DiagnosticsTab.tsx:276-289`, `SettingsScreen.tsx:179`. Inconsistent with the

- **M-U6. No confirmation before expensive re-index** — `DocumentsScreen.tsx:367-381, 602`. "Re-index

### Architecture

- **M-A1. TS↔shell-script config drift guarded by comments only** — `DRIVE_LAYOUT_DIRS`
  (`drive.ts:35-53`) is re-spelled in `prepare-drive.{ps1,sh}`; `drive_format_version: 1` is hardcoded
  in 5 spots; the `config/drive.json`/`policy.json` payloads + the sha256 regex + runtime/format gates
  are triplicated across scripts. The no-Node-on-a-fresh-machine constraint makes the *duplication*
  unavoidable, so the fix is *detection* — see H5's proposed drift test (covers this too).

- **M-A2. Chat/RAG streaming-handler duplication** — `registerChatIpc.ts:107-186` vs
  `registerRagIpc.ts:34-119` repeat the guard preamble + the entire stream lifecycle (AbortController,
  `inFlight.set`, token-send-with-`isDestroyed`-guard, done/error/finally) verbatim. Two copies of the
  most safety-sensitive path must be kept in lockstep by hand. Extract a
  `withChatStream(ctx, event, conversationId, runFn)` wrapper.

- **M-A3. `initBackend()` (~245 lines) and `AppContext` (12 fields) trending toward god-objects** —
  `main/index.ts`, `services/context.ts`. `initBackend` also has three copy-paste model resolvers
  (`resolveEmbeddingModel`/`resolveRerankerModel`/`resolveTranscriberModel`, L54-112). Collapse the
  resolvers into `resolveModelByRole(...)` and extract a `composeServices(...)` builder; acceptable at
  current size but watch the growth.

- **M-A4. `doctasks.ts` is 1582 lines mixing three task pipelines** (summary/translation/compare) with
  parallel-but-separate window-math/prompt blocks inviting subtle divergence. Split into
  `doctasks/{summary,translation,compare,manager}.ts` behind the existing `DocTaskManager` facade.

- **M-A5. No automated test spawns a real binary / loads real weights / hits real HTTP** (by design —
  the zero-binary/zero-network CI posture). The riskiest integration surface (real `spawn`, llama-server
  SSE, `--list-devices` parsing, WASM OCR) is covered only by the 15 `PAID_*` manual harnesses. Keep the
  posture, but document the `PAID_*` matrix as a **required pre-release gate** and add canned-real-output
  regression fixtures (SSE, `--list-devices`) whenever an upstream format is observed to change.

## LOW

### Security / correctness
- **L1.** Markdown link `href` scheme not whitelisted — `chat/Transcript.tsx:145`. A model could emit a
  `javascript:`/`data:` link; CSP + `setWindowOpenHandler` already block execution/navigation, so
  belt-and-suspenders only. Whitelist http(s) in the `a` renderer.
- **L2.** `cosineSimilarity` scores on `min(length)` instead of rejecting a mismatch —
  `embeddings/index.ts:61`. Defended at the only in-tree call site by the dimension guard; the exported
  contract ("equal-length") isn't enforced — a latent trap for future callers. Throw or return NaN on
  unequal lengths.
- **L3.** E5 batch reorder collapses missing `index` to 0 — `embeddings/e5.ts:171-184`. Mixed presence
  of `index` across response entries silently misaligns vectors↔chunks (count guard still passes).
  Require `index` on all entries or trust input order when none have it — never mix.
- **L4.** Embedder `suspend()` clears the failed-start latch *before* teardown —
  `embeddings/e5.ts:205-221`. A racing first-start that then fails re-arms `startFailed` after suspend
  cleared it, so the next `embed()` throws stale; user must lock/unlock twice. Clear `startFailed`
  *after* `teardown()`.
- **L5.** Transcriber `suspend()/stop()` don't await child exit — `transcriber/cli.ts:196-211`. On quit
  the parent may exit before whisper-cli dies, so the `finally` shred of the transient transcript JSON
  (default `workDir = tmpdir()`, never reached by the workspace crash-sweep) may not run — un-shredded
  transcript **content** in the OS temp dir, privacy-relevant. Await each child's `'close'`, or route
  transients under the swept workspace dir.
- **L6.** `rowToMessage` casts `citations_json` without shape validation — `chat.ts:133` (`as Citation[]`),
  while the sibling `parseScope` 6 lines up validates `Array.isArray` + per-element type. A structurally
  wrong-but-valid-JSON payload flows to the renderer. Validate the array shape.

  `chat/Transcript.tsx:116-123` — so `role="log"` either re-announces the full answer or goes silent for

  markdown remount), throttled to sentence boundaries.
- **L8.** Composer textarea has no accessible name — `chat/Composer.tsx:105-118` (placeholder only). Add
  `aria-label` (the PasswordField pattern).
- **L9.** Hardcoded English `docs` literal in the German UI — `HomeScreen.tsx:185` splices `<strong>docs

- **L10.** `String(e)` raw-error leaks bypass `friendlyIpcError` — `ChatScreen.tsx:195`,

- **L11.** Decorative `.spinner` spans not `aria-hidden` — `styles.css:697-709` + call sites (the
  DictationButton spinner is correctly hidden; the global one isn't). AT may announce an empty inline

- **L12.** ConfirmDialog body not associated via `aria-describedby` — `components/Dialog.tsx:62,116`

- **L13.** Password strength meter `role="status"` announces on every keystroke —

- **L14.** Search results region not announced / no result count — `chat/ConversationList.tsx:170-194`

- **L15.** Inline Thinking `<summary>` drives state via `preventDefault` and can desync the implicit

### Architecture / deps
- **L16.** The five `createSelected*` factory bodies are near-identical (~40 duplicated lines) —
  `{embeddings,reranker,transcriber}/factory.ts`. A shared `selectSidecarBacked(...)` helper would
  centralize the availability rule (the mock-vs-null divergence is the only meaningful difference).
- **L17.** `services/logging.ts` has zero automated tests — the only service with none, and it sits on
  every error path. Add a unit test for past-`MAX_BYTES` rotation + circular-`meta` non-throw.
- **L18.** `@napi-rs/canvas` (a native `.node` binary) is an optional transitive dep of `pdfjs-dist`,
  not imported in `src` — but it must **not** be bundled onto the portable drive (platform-specific,

- **L19.** `runtime/gpu.ts` `--list-devices`/Vulkan parser is tested only against canned strings (real

  Add a captured-real-output regression fixture.

## Suggested triage order

1. **H1, H4** — the two permanent session-wide wedges (one-line scope fixes each).
2. **H3, H2** — silent retrieval failures (one bad row kills all search; context overflow).
3. **M-C1/M-C2/M-C3** — sidecar lifecycle leaks + the self-disabling GPU auto-fallback.
4. **H5 + M-A1** — the one drift-detection vitest test (kills two config-drift findings at once).
5. **M-D1/M-D2/M-D3** — the BUILD_STATE §4 stale contract lines (test count, b9585, catalog counts).
6. **M-U1/M-U2 + L7/L8** — the highest-impact a11y/UX (error & streaming announcement, unlabeled input).
7. The remaining MEDIUMs (IPC arg guards, RRF tiebreak, abort plumbing, refactors) and LOWs as capacity allows.
